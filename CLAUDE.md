## Architecture

**Framework:** Next.js 14 App Router, TypeScript, Tailwind CSS, GSAP, Supabase.

### Route structure

| Route | Description |
|---|---|
| `/` | Landing page — all sections composed in `src/app/page.tsx` |
| `/blog` | Blog listing — server component, ISR (`revalidate = 60`), data from Supabase |
| `/blog/[slug]` | Blog post — server component, content rendered via `markdownToHtml` |
| `/work/[slug]` | Case study — server component, content from `public/Projects/<slug>/meta.md` |
| `/admin` | Login page (client component, `force-dynamic`) |
| `/admin/dashboard` | CMS dashboard — lists/toggles/deletes posts from Supabase |
| `/admin/editor/[id]` | Post editor — `@uiw/react-md-editor` |
| `/pricing` | Standalone pricing page |

### Data sources

- **Blog posts** — Supabase table `blog_posts` (columns: `id`, `title`, `slug`, `category`, `is_published`, `created_at`, `published_at`, `excerpt`, `content`). Blog pages use the generic `src/lib/supabase.ts` client; admin pages use `src/lib/supabase-browser.ts`.
- **Case studies / portfolio** — Markdown files at `public/Projects/<FolderName>/meta.md` with gray-matter frontmatter (`title`, `client`, `industry`, `year`, `vimeoId`, `tags`, `excerpt`) plus optional body content. Thumbnail at `public/Projects/<FolderName>/thumbnail.jpg`. Folder names are matched case-insensitively. Add a new project by creating a new folder with `meta.md` + `thumbnail.jpg`.

### Supabase clients — use the right one

- `src/lib/supabase.ts` — generic singleton (server-side data fetching in Server Components and blog pages).
- `src/lib/supabase-browser.ts` — `createBrowserClient` via `@supabase/ssr` (client components: admin pages).
- `src/lib/supabase-server.ts` — `createServerClient` with cookie forwarding (Server Components that need session-aware queries).
- `src/middleware.ts` — protects `/admin/dashboard` and `/admin/editor/*`; redirects unauthenticated users to `/admin`.

### Markdown pipeline

All markdown rendering goes through `src/lib/markdown.ts` (`markdownToHtml`):

```ts
remark().use(remarkGfm).use(remarkHtml, { sanitize: false })
```

This strips malformed double-prefixed headings (`## # Title` → `## Title`) before processing. **Do not use `react-markdown` in server components** — it is ESM-only and incompatible with Next.js 14 server components.

### GSAP animations

Every section that uses GSAP must follow this pattern:

```ts
useEffect(() => {
  const ctx = gsap.context(() => {
    // animations here
  }, sectionRef);
  return () => ctx.revert(); // CRITICAL — prevents memory leaks and ScrollTrigger conflicts
}, []);
```

Reference `Hero.tsx` or `Services.tsx` for examples.

### Design tokens

- Background: `#0A0A0A`
- Accent (cyan): `#00D4FF` — referenced as `text-accent`, `bg-accent`, `border-accent`
- Cards: `bg-white/[0.03]` + `border border-white/10` + `backdrop-blur-xl`
- Fonts: `DM Sans` (variable `--font-dm-sans`) and `DM Mono` (variable `--font-dm-mono`)
- Avoid standard borders; use glassmorphism + subtle glows (`shadow-[0_0_20px_rgba(0,212,255,0.3)]`).

### Navigation links

Always use absolute root-relative paths (e.g., `/#services`, `/#contact`) so anchor links work from sub-routes like `/blog/[slug]`.

### Voiceflow chat widget

Injected globally in `src/app/layout.tsx` via a `<Script strategy="afterInteractive">` block. Project ID: `69f1febf2081880a379990b9`.

### Sitemap

`public/sitemap-0.xml` is auto-generated by `next-sitemap` on every build. Do not edit it manually.
